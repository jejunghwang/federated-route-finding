# _docs/ — 작업 일지

코드 변경 시점의 **세션별 변경 내역**을 누적 기록한 폴더.

`docs/`(팀 공용 문서)와 별개로, 누가 언제 무엇을 왜 바꿨는지 추적하기 위해 사용한다. 폴더명에 `_` prefix를 붙여 공용 문서와 구분.

## 구조

```
_docs/
└── history/
    └── YYYY-MM-DD.md   # 그날 발생한 변경 내역만 기록 (프로젝트 개요 등 중복 정보 X)
```

## 기록 규칙

- 날짜.md에는 **그날 바뀐 것만** 자세하게 (파일명, Before/After, 이유)
- 같은 날 여러 변경이 있으면 같은 파일에 섹션 추가
- 변경 시점에 즉시 업데이트 (세션 끝에 몰아쓰지 않음)

## 다른 사람 참고용

이 폴더는 본인의 작업 일지로, 팀 워크플로우 가이드는 아님. **공식 가이드는 `docs/TEAM_WORKFLOW.md`** 참조.
